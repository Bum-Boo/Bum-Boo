# Bum-Boo

> 실제 사용자를 위한 local-first 생산성 도구, AI-assisted workflow system, 데스크톱 유틸리티를 만듭니다.

[Overview](../../README.md) | [English](README.en.md) | [한국어](README.ko.md) | [中文](README.zh-CN.md) | [日本語](README.ja.md)

Bum-Boo는 정보를 정리하고, 반복 작업을 자동화하며, 사용자가 자신의 도구와 데이터에 대한 통제권을 유지하도록 돕는 실용적인 소프트웨어에 집중합니다.

## 관심 영역

- 개인과 팀을 위한 local-first 자동화
- 사용자가 통제권을 유지하는 human-AI workflow 도구
- Windows와 creator workflow를 위한 데스크톱 생산성 유틸리티
- 비개발자를 위한 콘텐츠 운영 도구
- 상태가 명확하고 되돌릴 수 있으며 한계가 솔직하게 드러나는 HCI 중심 소프트웨어

## 대표 프로젝트

- [Vaultwright](https://github.com/Bum-Boo/Vaultwright): Codex/MCP workflow를 위한 Obsidian vault 유지관리 도구
- [BBCC](https://github.com/Bum-Boo/BBCC): 크리에이티브 단축키와 미디어 제어를 위한 Windows 컨트롤러 매핑 도구
- [BBTxT](https://github.com/Bum-Boo/BBTxT): 소스 코드, 로그, 설정 파일을 빠르게 찾는 로컬 텍스트 검색 도구
- [Naver-Cafe-Posts-Downloader](https://github.com/Bum-Boo/Naver-Cafe-Posts-Downloader): 네이버 카페 게시글 로컬 아카이브 매니저
- [BTS_sec](https://github.com/Bum-Boo/BTS_sec): 허가된 프로젝트를 위한 방어적 보안 점검 도구

## 작업 원칙

- 불필요한 계정, cloud sync, 숨겨진 데이터 이동을 줄입니다.
- 자동화는 범위가 명확하고 검토 가능해야 합니다.
- 비개발자도 실제 업무 흐름에서 쓸 수 있어야 합니다.
- 도구의 현재 한계와 실험 상태를 README에 직접 드러냅니다.
