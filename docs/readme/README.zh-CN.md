# Bum-Boo

> 为真实用户构建 local-first 生产力工具、AI 辅助工作流系统和桌面实用工具。

[Overview](../../README.md) | [English](README.en.md) | [한국어](README.ko.md) | [中文](README.zh-CN.md) | [日本語](README.ja.md)

Bum-Boo 关注实用软件：帮助用户管理信息、自动化重复流程，同时保持对工具和数据的控制权。

## 关注方向

- 面向个人和团队的 local-first 自动化
- 让用户保持控制权的 human-AI 工作流工具
- 面向 Windows 和创作者工作流的桌面生产力工具
- 面向非技术用户的内容运营工具
- HCI 导向的软件：清晰状态、可逆操作、明确边界和诚实的限制说明

## 代表项目

- [Vaultwright](https://github.com/Bum-Boo/Vaultwright): 面向 Codex/MCP 工作流的 Obsidian vault 维护工具
- [BBCC](https://github.com/Bum-Boo/BBCC): 用于创意快捷键和媒体控制的 Windows 控制器映射工具
- [BBTxT](https://github.com/Bum-Boo/BBTxT): 本地源码、日志和配置文件快速文本搜索工具
- [Naver-Cafe-Posts-Downloader](https://github.com/Bum-Boo/Naver-Cafe-Posts-Downloader): Naver Cafe 帖子本地归档管理器
- [BTS_sec](https://github.com/Bum-Boo/BTS_sec): 面向已授权项目的防御性安全审计工具

## 工作原则

- 减少不必要的账号、云同步和隐藏的数据移动。
- 自动化必须有清晰范围，并且便于用户检查。
- 工具应服务真实工作流，而不仅仅是演示效果。
- 项目的限制、实验状态和安全边界应直接写清楚。
